User input module

This Python node represents the action client node that deals with asking the user to enter the coordinates (x, y), or to cancel them, that the robot has to reach. After that, it publishes the robot position and velocity as a custom message on the /Position_velocity topic, relying on the values of the /odom topic.

scripts.user_input.client()[source]

This function handles the action client functionality.In particular, it creates an action client for the ‘/reaching_goal’ action server, waits for the server to start, and then enters a loop to prompt the user for the x and y coordinates of the goal position. It creates a goal message of type PlanningGoal with the provided coordinates, sends the goal to the server, and waits for a response. The user has 5 seconds to cancel the goal by typing ‘c’, and if the cancellation is requested, the function cancels the goal and prints a cancellation message.

Parameters:

None –

Returns:

None

scripts.user_input.main()[source]

Main function initializes the ROS node with the name ‘user_input’, sets up a publisher to send a message containing velocity and position parameters, and subscribes to the ‘/odom’ topic to receive velocity and position parameters from the ‘Odometry’ message. It then calls the ‘client’ function to handle the action client functionality for sending goals to the ‘/reaching_goal’ server.

Service module

This node represents the service node. Moreover, when the node is called, it prints the number of successfull reaching desired position and the number of cancelling goals.

scripts.service.data(req)[source]

This function is a callback function for the service server. It receives a request and returns a response containing the values of goal_reach and goal_canc as a goal_srvResponse object.

Parameters:

req (goal_srvRequest) – The received goal_srvRequest object representing the request.

Returns:

The response containing the values of goal_reach and goal_canc.

Return type:

goal_srvResponse

scripts.service.main()[source]

This function initializes the ROS node with the name ‘service’, creates a service with the name ‘service’ and the type ‘goal_srv’, sets up a subscriber to receive messages of type ‘PlanningActionResult’ from the ‘/reaching_goal/result’ topic, and then enters a loop to wait for messages. The function will keep running until the ROS node is shutdown.

scripts.service.result(msg)[source]

This function is a callback function for the result subscriber. It receives a message containing the status of the result. Based on the status value, it updates the global variables goal_canc and goal_reach to keep track of the number of cancelled and reached goals.

Information module

The aim of this node is to print, on the console, the distance of the robot from the desired position and its average speed. These parameters are taken from the /Position_velocity topic as a custom message. The information is printed with the frequency set as a parameter in the launch file.

scripts.print_info.main()[source]

Main function initializes the ROS node with the name ‘print_info’, retrieves the publish frequency parameter from the ROS parameter server, sets up a subscriber to receive messages of type ‘Position_velocity’ from the ‘/pos_vel’ topic, and then enters a loop to wait for messages. The function will keep running until the ROS node is shutdown.

Parameters:

None –

Returns:

None

scripts.print_info.posvel(msg)[source]

This function is called when a message is received from the info subscriber. It computes the time period based on the given frequency, gets the desired position and actual position from the message and calculates the distance between them. It also computes the average speed and prints the information if the specified time period has elapsed since the last print.

Parameters:

msg (Position_velocity) – The received Position_velocity message containing position and velocity information.

None
